The 27 August 2025 release introduces aggregation, a new extraction mode that synthesises answers from multiple independent sections of a document.

## Aggregation

Vault now supports a new type of extraction called Aggregation. Vault will use multiple, independent sections of a document to answer questions.

Aggregation is useful for "list all X" queries or open-ended questions such as "What is the effective date?" when the answer can be inferred from multiple independent sections — for example, the definition, signatures, and date of last signature.

Aggregation is available for all tags. To set a tag as an aggregation, simply enable the aggregation option when you create it.

![Aggregation option enabled on a tag](https://downloads.intercomcdn.com/i/o/h2osv3w3/1695750447/dc0a76667adda264b00c36a2ee06/Screenshot+2025-08-27+at+14_44_30.png)

The answer will then display the aggregation symbol in the Vault tag card.

![Vault tag card showing the aggregation symbol](https://downloads.intercomcdn.com/i/o/h2osv3w3/1695754666/fcd91e6361c03348eff3b839e718/Screenshot+2025-08-27+at+14_46_30.png)

Aggregation tags can be created for default, custom, and Vault team tags.
