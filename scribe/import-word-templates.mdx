If your organisation already has document templates in Microsoft Word, you don't need to re-type them in Scribe. You can import a `.docx` file directly and Scribe will convert it into an editable template, preserving your existing content and formatting so you can get up and running faster.

## Before you start

Make sure your Word document is saved in `.docx` format. During the import process you'll assign the template a folder and type, so it helps to know in advance where you want to store it and what category it belongs to.

## Import steps

<Steps>
  <Step title="Start a new template">
    From the Scribe dashboard, click **Create template**. This starts the same creation flow used for building templates from scratch.
  </Step>
  <Step title="Choose a folder">
    Select the folder where this template will live. You can pick an existing folder or create a new one. Folders let you group templates by document category, counterparty, or team.
  </Step>
  <Step title="Set the document type">
    Choose the type for your template — for example, "NDA" or "Service Agreement". If you're creating a new type, you'll be asked to confirm some additional details about it.
  </Step>
  <Step title="Upload your Word file">
    On the next screen you'll see the option to import a file. You can either drag and drop your `.docx` file onto the upload area, or connect to a supported cloud file storage system to select the file from there.
  </Step>
  <Step title="Review the converted template">
    Once the upload is complete, Scribe converts the Word document into a Scribe template. Open the template to review the content and begin adding fields, conditional logic, or any other customisations you need.
  </Step>
</Steps>

<Tip>
After importing, take a moment to review the converted template carefully. Complex Word formatting — such as nested tables or custom styles — may need minor adjustments in the Scribe editor to look exactly as intended.
</Tip>

## What happens after import

Your imported file becomes a fully editable Scribe template. From here, everything works the same as a template you built from scratch:

- Insert fields to mark the variable parts of the document
- Add conditional logic for sections that depend on user choices
- Invite teammates to comment, request changes, or flag issues
- Publish to Legislate teams when the template is ready

<Note>
Importing a Word file does not automatically create fields. You'll need to highlight the relevant text in the editor and assign fields manually after the import.
</Note>
