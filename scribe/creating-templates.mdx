Creating a template in Scribe gives your team a reusable, form-driven document that anyone with the right access can use to generate correctly structured agreements in minutes. This page walks you through the entire process, from the initial setup to handing off a draft for collaborative review.

## Create a new template

To get started, click **Create template** from the Scribe dashboard.

<Steps>
  <Step title="Choose a folder">
    Specify the folder where the template will be stored. Folders help you group templates by type, counterparty, or business unit. You can select an existing folder or create a new one on the spot.
  </Step>
  <Step title="Set the document type">
    Choose the type for your new template — for example, "NDA" or "Employment Contract". Like folders, you can select an existing type or define a new one. If you create a new type, you'll be prompted to confirm a few additional details about it.
  </Step>
  <Step title="Open the editor and start drafting">
    Once the template is created, click on it to open the Scribe editor. You can format text, adjust styles, and write the document body just as you would in a standard word processor.
  </Step>
</Steps>

## Insert fields into the template

Fields are the variables in your template — the parts that will differ each time a document is created from it (such as a party name, a date, or a fee amount). To insert a field:

1. Highlight the portion of text in the editor where the field value should appear.
2. The toolbar will offer an option to add a new field or select an existing one.
3. Configure the field's name, type, label, validation, and group (see [Template Fields](/scribe/template-fields) for a full reference).
4. Save the field. It will appear in the fields panel on the right side of the editor, highlighted in the document text.

<Tip>
The **label** is the question users will see when filling in the form to create a document. Write it as a clear, natural-language question — for example, "What is the start date of the agreement?" rather than "Start Date".
</Tip>

Fields you create in one template are available to reuse in other templates, saving you time when working across similar document types.

## Manage existing fields

You can view and edit all the fields in your template at any time by clicking the **settings icon** in the fields panel on the right. From there you can:

- Edit a field's name, label, type, validation, or group
- Delete a field that is no longer needed
- Review which fields are already mapped to text in the document

<Warning>
Once a template has been published, you can no longer edit or delete its fields. Make sure your fields are finalised before publishing.
</Warning>

## Comments, requests, and flags

Scribe includes a full collaborative drafting workflow so your team can review and improve templates together.

<Tabs>
  <Tab title="Edit mode">
    In Edit mode, changes you make are applied directly and immediately to the template. Use this mode when you have permission to make authoritative edits.
  </Tab>
  <Tab title="Request mode">
    In Request mode, your changes are tracked rather than applied outright. Each change becomes a request that a teammate with editing rights can accept or reject. Switch to Request mode by clicking the mode button in the top-left ribbon, between the Publish and Save buttons.

    A requester cannot accept their own requests.
  </Tab>
  <Tab title="Comments">
    Highlight any portion of text and enter a comment from the toolbar. Comments are grouped under the **Comments** tab in the right panel. Use comments to ask questions or flag points for discussion without proposing a specific wording change.
  </Tab>
  <Tab title="Flags">
    Flags let you escalate a section of the draft to the wider team. Select the text, choose the flag option in the toolbar, and the flag will appear under the **Flags** tab.
  </Tab>
</Tabs>

<Note>
You cannot publish a template that has unresolved comments, requests, or flags. All open items must be resolved first.
</Note>

To invite teammates to collaborate on a template, click the **pen icon** in the top-right corner of the editor.
