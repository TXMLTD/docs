Vault extracts as many tags as possible based on the document type, but you may occasionally need a data point that is not part of Vault's standard tag set for that document. Manual fields let you create your own custom tags by selecting text directly from the document.

## When to use manual fields

Use a manual field when:

- A specific clause or value is important to your business but not covered by Vault's standard tags for the document type.
- You want to capture a unique piece of information that varies only in this particular document.

<Tip>If you find yourself creating the same manual field across many documents of the same type, consider contacting TextMine support at support@textmine.com to request it as a standard tag for that document type.</Tip>

## How to create a manual tag

<Steps>
  <Step title="Open the document in the Vault editor">
    From the Vault home screen, click the pen icon on the document you want to edit.
  </Step>
  <Step title="Switch to plain text view">
    In the editor, switch from PDF view to the **plain text view** using the view toggle. The document's text content is displayed as plain, selectable text.
  </Step>
  <Step title="Highlight the relevant text">
    Hover over and then select the text that corresponds to the tag you want to create. A tooltip or action prompt will appear.
  </Step>
  <Step title="Create the tag">
    Click **Create field** (or the equivalent action shown in the prompt). The new tag appears under the **Created fields** tab in the right-hand panel.
  </Step>
  <Step title="Label the field">
    Give the field a meaningful name so it is identifiable when the document is submitted to Legislate.
  </Step>
</Steps>

## Viewing your manual tags

Manual tags appear in the **Created fields** tab in the right-hand panel, separate from the AI-extracted tags. You can validate and submit them in the same way as any other tag.

<Note>Manual fields are unique to the document they were created in. They do not automatically apply to other documents of the same type.</Note>
